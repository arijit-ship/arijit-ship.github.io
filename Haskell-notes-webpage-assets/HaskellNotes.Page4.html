10. Random Numbers

Before we start coding, first thing‚Äôs first. What is a random number?

A random number is a number chosen as if by chance from some specified distribution such that selection of a large set of these numbers reproduces the underlying distribution. Almost always, such numbers are also required to be independent, so that there are no correlations between successive numbers.

So here‚Äôs the long story short: Random numbers are just unpredictable numbers and you can not correlate two successive random numbers.

  • How does a computer generate a random number when you write a program to generate one?

    Computers can generate truly random numbers by observing some outside data, like mouse movements or fan noise, which is not predictable, and creating data from it. This is known as entropy. Other times, they generate ‚Äúpseudorandom‚Äù numbers by using an algorithm so the results appear random, even though they aren‚Äôt. [ howtogeek.com ]

  • How random they are?

    As said above, they can be almost random depending upon how they are being generated. However, "pseudorandom" numbers are made possible by the algorithm. These pseudorandom numbers are not truly random numbers, and as they are generated by an algorithm, they can be predicted.

A pseudorandom number generator, also known as a deterministic random bit generator, is an algorithm for generating a sequence of numbers whose properties approximate the properties of sequences of random numbers.

Pseudorandom numbers are useful for experiments and simulations, however, they‚Äôre not used for security purposes.

11. Head or Tail ‚Äî Random Numbers

We are flipping 10 coins simultaneously and counting the number of heads and tails.

The code

import System.Random
tosses :: (Double,Double) -> StdGen -> Double  -> Int -> (Int,Int)
tosses (lowlim,uplim) rgen cutoff numtosses = (heads, numtosses-heads)
 where
  heads = length $ filter (>= cutoff) $ take numtosses $ (randomRs (lowlim,uplim) rgen :: [Double])

main = do
 rgen <- newStdGen
 print $ tosses (0,1.0) rgen 0.5 10

So how are we counting the number of heads and tails using random numbers?
Well, here we're generating 10 random numbers (for flipping 10 coins simultaneously or flipping only one coin 10 times), each between 0 to 1.0, if the outcome is >0.5, we will be calling it a head otherwise it‚Äôll be a tail. This particalur value >0.5 is the cutoff.

12. Some more on Random Numbers

  • Random seed (wikipedia.org)

A random seed (or seed state, or just seed) is a number (or vector) used to initialize a pseudorandom number generator.
For a seed to be used in a pseudorandom number generator, it does not need to be random. Because of the nature of number generating algorithms, so long as the original seed is ignored, the rest of the values that the algorithm generates will follow probability distribution in a pseudorandom manner.
A pseudorandom number generator‚Äôs number sequence is completely determined by the seed: thus, if a pseudorandom number generator is reinitialized with the same seed, it will produce the same sequence of numbers.
The choice of a good random seed is crucial in the field of computer security. When a secret encryption key is pseudorandomly generated, having the seed will allow one to obtain the key. High entropy is important for selecting good random seed data.

# Comments
System.Random This library deals with the common task of pseudo-random number generation. The library makes it possible to generate repeatable results, by starting with a specified initial random number generator, or to get different results on each run by using the system-initialised generator or by supplying a seed from some other source.
StdGen The System.Random library provides one instance of RandomGen, the abstract data type StdGen.
newStdGen Applies split to the current global random generator, updates it with one of the results, and returns the other.
randomR randomR :: (RandomGen g, UniformRange a) => (a, a) -> g -> (a, g). Takes a range (lo,hi) and a pseudo-random number generator g, and returns a pseudo-random value uniformly distributed over the closed interval [lo,hi], together with a new generator. It is unspecified what happens if lo>hi. For continuous types there is no requirement that the values lo and hi are ever produced, but they may be, depending on the implementation and the interval.
randomRs randomRs :: RandomGen g => (a, a) -> g -> [a]. Plural variant of randomR, producing an infinite list of pseudo-random values instead of returning a new generator.
